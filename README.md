# Testability Scanner

**Mobile Accessibility ID Reporter** — v4.5

A desktop tool that connects to a running mobile application via Appium, collects all interactive UI elements, classifies their accessibility IDs, generates AI-powered naming suggestions, and produces Word / Excel / JSON reports.

## What It Does

Testability Scanner connects to a live mobile application session through Appium and scans every interactive element on the current screen. Each element is classified into one of four ID statuses:

| Status | Meaning |
|---|---|
| ✅ **Unique ID** | Element has a valid, unique accessibility ID |
| ⚠️ **Undefined ID** | Element has an ID containing "undefined" or starting with `__` |
| 🔁 **Duplicate ID** | Same accessibility ID is used by more than one element |
| ❌ **Missing ID** | Element has no accessibility ID at all |

For elements that are missing or have invalid IDs, Claude AI generates a suggested snake_case name following the app's existing naming conventions. Results are exported to Word, Excel, and/or JSON.

## Configuration

All settings are stored in **`gui_config.json`** and managed through the GUI. You do not need to edit this file manually.

`config.py` is auto-generated by the application before each scan run. Do not edit it directly — any manual changes will be overwritten.

### Device Profiles

Multiple device profiles can be saved for both iOS and Android. Each profile stores:

**iOS profile fields:**
```json
{
  "device_name": "iPhone 16",
  "platform_version": "18.6",
  "bundle_id": "com.example.myapp",
  "udid": "AD21A917-5271-4DF1-8C5D-E64A0DE8EAD9"
}
```

**Android profile fields:**
```json
{
  "device_name": "emulator-5554",
  "platform_version": "13",
  "app_package": "com.example.myapp",
  "app_activity": "com.example.myapp.MainActivity"
}
```

### Blacklist IDs

System-generated resource IDs that should be ignored can be listed in the **Blacklist IDs** field (comma-separated). Common defaults:

```
statusBarBackground, content, action_bar_root, navigationBarBackground,
LinearLayout, FrameLayout
```

### Document Sections

Choose which element categories appear in the generated reports. Sections can be toggled independently:

- Unique ID
- Undefined ID
- Duplicate ID
- Missing ID

## API Key Setup

The Claude AI features (element filtering fallback and ID suggestion generation) require an Anthropic API key.

**Option 1 — Environment variable (recommended):**
```bash
export ANTHROPIC_API_KEY="sk-ant-..."
```

**Option 2 — Key file:**

Create a file named `.anthropic_key` in the project root and paste your key inside:
```
sk-ant-api03-...
```

> ⚠️ Add `.anthropic_key` and `config.py` to your `.gitignore` to avoid committing secrets.

If no API key is provided, the application falls back to coordinate-based element matching only. AI suggestion and vision fallback features will be disabled.

## Scan Modes

### 1 — Full Scan (📋 Full Scan)

Connects to the app, scans the current screen once, and immediately generates a report.

**Steps:**
1. Select platform (iOS / Android)
2. Choose or create a device profile
3. Set output folder and format
4. Click **▶ RUN**
5. Enter the page name when prompted in the console
6. Reports are saved to the output folder

**Use when:** You want a complete snapshot of a single screen.

---

### 2 — Smart Scan (🎯 Smart Scan)

Takes a screenshot first, lets you draw annotation boxes over the areas of interest, then limits the element scan to only those areas.

**Steps:**
1. Configure platform and profile
2. Click **📱 CONNECT & CAPTURE**
3. In the annotation window, draw rectangles around the elements to include
4. Click **✓ Confirm → Generate Report**
5. Enter the page name in the panel that appears

**How matching works:**
- Elements whose center coordinates fall inside a drawn box are included directly
- Boxes that have no coordinate match trigger a Claude Vision API call to identify elements visually

**Use when:** A screen has too many elements and you only want to report on a specific section.

---

### 3 — Session Scan (📱 Session Scan)

Keeps the Appium driver alive across multiple page collections. Scans accumulate into a single multi-page report.

**Steps:**
1. Enter a **flow name** (e.g. `login_flow`, `checkout`)
2. Click **🔌 START SESSION**
3. Navigate to the first page in the app
4. Click **📥 COLLECT PAGE** — repeat for each page
5. When done, click **✅ DONE - GENERATE REPORT**

A summary table in the right panel shows element counts per collected page in real time.

**Use when:** Reporting across a multi-page user flow (e.g. onboarding, checkout).

---

### Build Summary (📊 Merge Sheets)

Reads an existing Excel report file and generates three additional sheets:

| Sheet | Content |
|---|---|
| **Data** | Flat list of all elements from every page sheet |
| **Summary** | Page-by-page breakdown with status counts and percentages |
| **Task** | Only elements with `Missing / Undefined / Duplicate` status; formula-linked to source sheets |

To run: select an Excel file using the file picker in the footer, then click **📊 Merge Sheets**.

## Output Files

### Word Document (`.docx`)

A table listing every scanned element with columns:

`Element ID | Page | Type | Label / Text | Value | Accessibility ID / Resource ID | Status | New Status | AI Suggestion`

Followed by the screenshot at the end of the document.

### Excel Workbook (`.xlsx`)

Same columns as Word, colour-coded by status. Each page name becomes its own sheet. Screenshots are embedded in a dedicated image column. Run **Merge Sheets** afterwards to add Data, Summary, and Task views.

### JSON (`.json`)

Contains only `Unique ID` elements for which an AI suggestion was successfully generated:

```json
[
  {
    "key": "loginEmailTextbox",
    "androidValue": "login_email_input",
    "androidType": "id",
    "iosValue": "login_email_input",
    "iosType": "accessibilityId"
  }
]
```

## Internationalization

The application supports Turkish (TR) and English (EN). Language can be switched at runtime from the top-right dropdown — no restart required.

All user-facing strings are defined in `strings.json`. To add or update text:

1. Add the key under both `"TR"` and `"EN"` sections in `strings.json`
2. Reference it in code with `t("your_key")` or `t("your_key", param=value)`

**Convention for new keys added with code changes:**

```json
"TR": {
  "your_new_key": "Türkçe metin {param}",
  ...
},
"EN": {
  "your_new_key": "English text {param}",
  ...
}
```

If a key exists in neither language, `t()` returns the key name itself as a silent fallback — this is visible in the UI and should be treated as a bug.

## Ne Yapar

Testability Scanner, Appium üzerinden canlı bir mobil uygulama oturumuna bağlanır ve ekrandaki tüm etkileşimli elementleri tarar. Her element dört ID durumundan birine sınıflandırılır:

| Durum | Açıklama |
|---|---|
| ✅ **Unique ID** | Elementin geçerli ve benzersiz bir erişilebilirlik ID'si var |
| ⚠️ **Undefined ID** | Elementin ID'si "undefined" içeriyor veya `__` ile başlıyor |
| 🔁 **Duplicate ID** | Aynı erişilebilirlik ID'si birden fazla element tarafından kullanılıyor |
| ❌ **Missing ID** | Elementin hiç erişilebilirlik ID'si yok |

Eksik veya geçersiz ID'ye sahip elementler için Claude yapay zekası, uygulamanın mevcut isimlendirme kurallarını takip eden snake_case önerileri üretir. Sonuçlar Word, Excel ve/veya JSON olarak dışa aktarılır.

## Tarama Modları

### 1 — Tam Tarama (📋 Full Scan)

Uygulamaya bağlanır, mevcut ekranı tek seferde tarar ve hemen bir rapor oluşturur.

**Adımlar:**
1. Platform seçin (iOS / Android)
2. Bir cihaz profili seçin veya oluşturun
3. Çıktı klasörünü ve formatını belirleyin
4. **▶ ÇALIŞTIR** butonuna tıklayın
5. Konsolda sorulduğunda sayfa adını girin
6. Raporlar çıktı klasörüne kaydedilir

**Ne zaman kullanılır:** Tek bir ekranın tam anlık görüntüsünü almak istediğinizde.

---

### 2 — Akıllı Tarama (🎯 Smart Scan)

Önce ekran görüntüsü alır, ilgilenilen alanlara annotation kutuları çizmenizi sağlar, ardından element taramasını yalnızca bu alanlarla sınırlandırır.

**Adımlar:**
1. Platform ve profili yapılandırın
2. **📱 BAĞLAN & GÖRÜNTÜ AL** butonuna tıklayın
3. Açılan annotation penceresinde dahil edilecek elementlerin etrafına dikdörtgen çizin
4. **✓ Onayla → Raporu Oluştur** butonuna tıklayın
5. Beliren panelde sayfa adını girin

**Eşleştirme nasıl çalışır:**
- Merkez koordinatları çizilen bir kutunun içine düşen elementler doğrudan dahil edilir
- Koordinat eşleşmesi bulunamayan kutular için Claude Vision API'si devreye girerek elementleri görsel olarak tanımlar

**Ne zaman kullanılır:** Ekranda çok fazla element olduğunda ve yalnızca belirli bir bölümü raporlamak istediğinizde.

---

### 3 — Oturum/Akış Tarama (📱 Session Scan)

Appium sürücüsünü birden fazla sayfa koleksiyonu boyunca açık tutar. Taramalar tek bir çok sayfalı raporda birikir.

**Adımlar:**
1. Bir **akış adı** girin (örn. `giris_akisi`, `odeme`)
2. **🔌 OTURUMU BAŞLAT** butonuna tıklayın
3. Uygulamada ilk sayfaya gidin
4. **📥 SAYFAYI TOPLA** butonuna tıklayın — her sayfa için tekrarlayın
5. Bitince **✅ BİTTİ - RAPOR OLUŞTUR** butonuna tıklayın

Sağ paneldeki özet tablo, toplanan her sayfa için element sayılarını gerçek zamanlı olarak gösterir.

**Ne zaman kullanılır:** Çok sayfalı bir kullanıcı akışında raporlama yaparken (örn. onboarding, ödeme süreci).

## Çoklu Dil Desteği

Uygulama Türkçe (TR) ve İngilizce (EN) dillerini destekler. Dil, sağ üst köşedeki açılır menüden çalışma zamanında değiştirilebilir — yeniden başlatma gerekmez.

Tüm kullanıcıya gösterilen metinler `strings.json` dosyasında tanımlanmıştır. Metin eklemek veya güncellemek için:

1. Anahtarı `strings.json` içinde hem `"TR"` hem de `"EN"` bölümüne ekleyin
2. Kodda `t("anahtar_adi")` veya `t("anahtar_adi", param=deger)` ile kullanın

**Kod değişiklikleriyle eklenen yeni anahtarlar için kural:**

```json
"TR": {
  "yeni_anahtar": "Türkçe metin {param}",
  ...
},
"EN": {
  "yeni_anahtar": "English text {param}",
  ...
}
```

Bir anahtar her iki dilde de yoksa `t()` anahtarın kendisini sessiz fallback olarak döndürür — bu arayüzde görünür olur ve hata olarak değerlendirilmelidir.
